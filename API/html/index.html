get /lambda
Return Function list (lambdaGet)
The Function which belongs to a specific user is displayed

Consumes

This API call consumes the following media types via the Content-Type request header:
  • multipart/form-data

Query parameters

\UserName (required)
Query Parameter — Name of the person whose files are to be displayed

Return type

array[lambda]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Returns a json array with all the functions uploaded by the user and metadata

0

Unexpected error

put /lambda
Update an already existing function (lambdaPut)
Updates the function and other metadata based on the input parameters UserName and Filename

Consumes

This API call consumes the following media types via the Content-Type request header:
  • multipart/form-data

Query parameters

userName (required)
Query Parameter — Username of the person accessing it
eventType (required)
Query Parameter — Type of event that the lambda function responds to
eventSource (required)
Query Parameter — Type of event that the lambda function responds to
memory (required)
Query Parameter — Minimum memory requirement of the lambda function
data (required)
Query Parameter — File to be uploaded
environment (required)
Query Parameter — Running environment of the lambda function

Return type

array[lambda]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Updates the function and the metadata and returns a json object of updated values

0

Unexpected error

delete /lambda
Delete a function (lambdaDelete)
The function name for a specific user given in the parameters is deleted

Consumes

This API call consumes the following media types via the Content-Type request header:
  • multipart/form-data

Query parameters

fileName (required)
Query Parameter — Name of the lambda file to be deleted (.py)
userName (required)
Query Parameter — Name of the person who owns the lambda function

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Displays a message after the function has been successfully deleted or an error message if the function name entered does not exist
Up
